background · motivation

프로젝트 배경

수작업 반복 작업

레고 조립은 반복 작업과 높은 위치 정밀도가 요구됨. 수동으로는 효율적으로 처리하기 어려운 대량 조립.

단순 Pick & Place의 한계

이미지 설계 → 블록 분할 → 체결 검증까지의 전 과정을 자동화하기 어려움.

실환경 문제

조명, 카메라 왜곡, 블록 누락, 체결 실패, 외력 충돌 등 실제 환경에서 다양한 문제 발생.

통합 폐루프 시스템 필요

"비전 → 계획 → 로봇 → 검증 → UI"를 하나의 폐루프 시스템으로 통합.

sequencer · problem

블록 분할 문제

이미지 분석 결과는 셀 단위 색상 배열이지만,
실제 로봇은 2칸 또는 3칸 블록을 집어야 합니다.

같은 색상 8칸 연속 → 여러 분할 가능
3 + 3 + 2
3 + 2 + 3
2 + 3 + 3
2 + 2 + 2 + 2

왜 분할이 중요한가

!잘못 분할하면 구조 안정성 하락
!위층 블록 경계가 아래층 경계와 겹치면 불안정
!Greedy 방식은 구조를 고려하지 못함
✓→ 최적 분할 알고리즘 필요

sequencer · algorithm

Beam Search DP 설계

1각 run을 2와 3의 순서 있는 조합으로 분할
2아래층부터 후보 생성 — 기초 레이어 먼저 확정
3아래층 내부 경계를 계산
4위층 블록이 아래층 경계를 적절히 걸치는지 평가
5위반 시 loss 부여, 누적 loss가 낮은 후보 유지
6beam width로 후보 수를 제한해 계산량 제어

누적 loss 최소화
🎯

design · decision

왜 Greedy가 아니라 Beam Search인가

❌ Greedy

빠르지만 구조 안정성을 전혀 고려하지 못함. 첫 번째 분할만 보고 결정.

✅ Beam Search

후보 수를 제한해 계산량 제어. Greedy보다 좋은 구조 후보 탐색. 속도·품질 균형.

⚠️ 완전 탐색

가능한 모든 조합을 보지만 실행 시간이 커짐. 실시간 데모에 부적합.

결론: 실시간성, 구현 난이도, 안정성 사이의 절충안으로 Beam Search DP가 최적.

robot · coordinate

로봇 좌표 변환

Y 좌표 변환 (격자 col → 로봇 Y)

y = y_col0 - center_col × cell_pitch
center_col = col_start + (width−1) / 2

Z 높이 변환 (layer → 배치 Z)

z = base_z + layer × stack_pitch
stack_pitch = 19.0 mm + 1.0 mm = 20.0 mm

변환 파라미터

col격자 열 인덱스 → Y 좌표
row격자 행 인덱스 → layer
layer조립 높이 Z 계산에 사용
block_type2칸 / 3칸 블록 구분
color키팅 트레이 위치 선택

troubleshooting · 01 / 05

주요 트러블 1: ROS2 Deadlock 위험

PROBLEM

action callback 안에서 service future를 잘못 기다리면 deadlock 가능. 로봇 실행이 완전히 멈춤.

SOLUTION

웹캠 service client를 별도 helper node와 executor로 분리. future 완료는 threading.Event로 대기.

선택 이유

구조가 복잡해지지만 로봇 실행 callback과 service response 처리를 완전히 분리. 실제 실행 중 멈춤 현상 예방.

troubleshooting · 02 / 05

주요 트러블 2: 초록색 인식 불안정

PROBLEM

조명과 카메라 조건에 따라 green이 배경 또는 메인보드 색상과 혼동. green 블록 미검출 또는 오검출 발생.

SOLUTION

green 전용 LAB threshold 확대. HSV fallback mask 추가. green_min_color_fraction 별도 설정.

선택 이유

전체 threshold를 높이면 다른 색상 오검출 증가. green만 별도 보정하는 것이 부작용이 가장 작음.

troubleshooting · 03 / 05

주요 트러블 3: 블록 누락으로 인한 허공집기

PROBLEM

트레이에 블록이 없으면 그리퍼가 허공을 집음. 이후 place 단계에서 아무것도 없는 블록을 조립하려다 전체 sequence 실패.

SOLUTION

pick 직전 웹캠 검사 실패 시 pause. 사용자가 블록 보충 후 resume하면 해당 task 재시도.

선택 이유

즉시 abort보다 데모 운영성이 좋음. 허공집기 이후 place 실패를 사전에 차단해 연쇄 실패 방지.

주요 트러블 4: 조립 위치 미세 오차

PROBLEM

레고 체결은 작은 XY/Z 오차에도 실패 가능. 고정 Z 하강만으로는 체결 성공률이 낮음.

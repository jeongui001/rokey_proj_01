# Sequencer 노드 역할 정의

## 0. 2차원 격자 생성

- 240개 색상 배열(1차원, row-major)을 24×10 2차원으로 변환 (가로 24, 세로 10)
- row 0 = 이미지 상단, row 9 = 이미지 하단
- 이 2차원 격자를 현실의 Y축(가로) × Z축(세로)에 세움
- X축 좌표는 고정값
- Z축 좌표는 힘 탐지로 배치하므로 계산 불필요

## 1. 색상 런 추출

행 단위로 동일 색상 인접 셀을 묶어 런(run)을 생성한다.

- 빈 셀(`""`)과 empty 팔레트(`"empty"`)는 건너뜀
- 길이 1인 런은 블록을 만들 수 없으므로 건너뜀 (해당 셀에 블록 미배치)

## 2. 블록 유형 매칭 (Beam Search DP)

셀 단위가 블록이 아니라, 셀들이 모여서 블록을 구성한다.

| 유형 | 가로 (셀) | 세로 (셀) |
|------|----------|----------|
| 유형 1 | 2 | 1 |
| 유형 2 | 3 | 1 |

### 분할 규칙

각 런을 2와 3의 순서 있는 조합으로 분할한다. 순서가 다르면 별개의 분할이다.

점화식: `f(n) = f(n-2) + f(n-3)` (f(0)=1, f(1)=0, f(2)=1, f(3)=1)

### 벽돌 패턴 규칙

위 행의 블록이 아래 행 블록 경계를 걸치도록 배치하여 구조적 안정성을 확보한다.

**경계 정의**: 아래 행 블록 분할의 내부 구분점 (행 양쪽 끝 0, 24는 제외)

**판정**: 윗 블록 [a..b] 범위 [a, b+1] 안에 포함되는 아래 경계 수를 센다.

| 조건 | 결과 |
|------|------|
| 경계 ≥ 2개 | 탈락 |
| 경계 1개, 윗 블록의 끝(a 또는 b+1)과 겹침 | 탈락 |
| 경계 1개, 윗 블록 내부에 있음 | 합격 |

### Beam Search DP

- beam_width = 5 (파라미터)
- 맨 아래 행 (layer 0): 유형2 블록 수 내림차순으로 상위 5개 선택 (loss = 0)
- 나머지 행 (layer 1~9): 이전 5개 후보 × 현재 행 분할 후보
  - 블록별 벽돌 규칙 위반 시 `+(10 - layer)` loss (아래 행일수록 큰 페널티)
  - 누적 loss 기준 상위 5개 유지
- 최종: 최소 loss 후보 선택

## 3. 배치 순서

- 이미지 좌하단부터 시작 (row 9 → row 0)
- 블록 단위로 처리: 한 줄 가로로 쌓고 다음 줄로 이동
- 빠른 적층을 위해 지그재그(boustrophedon) 순서
  - 짝수 layer (0, 2, 4, ...): 좌 → 우
  - 홀수 layer (1, 3, 5, ...): 우 → 좌

## 4. 배치 위치 계산

### 시작점

맨 오른쪽 블록의 Y좌표. 그리드 끝(col 24)에서 떨어진 만큼 보정한다.

| 블록 유형 | 기준 좌표 (x, y) |
|----------|-----------------|
| 유형 1 | (332, 310) |
| 유형 2 | (332, 302.05) |

```
right_gap = 24 - (블록 시작 col + 블록 폭)
Y = 기준 Y - right_gap × 15.9
```

### Y축 이동량 (우→좌 순회 시)

이전 블록(오른쪽)과 현재 블록(왼쪽)의 유형 조합에 따라 이동량이 달라진다.

| 이전 유형 → 현재 유형 | Y 이동량 |
|---------------------|---------|
| 1 → 1 | 15.9 × 2 = 31.8 |
| 2 → 1 또는 1 → 2 | 15.9 × 2.5 = 39.75 |
| 2 → 2 | 15.9 × 3 = 47.7 |

### 빈칸 보정

블록 사이에 빈 셀(empty 또는 길이 1 런)이 있으면, 빈 셀 수 × 15.9를 이동량에 추가한다.

```
gap = 다음 블록 시작 col - (현재 블록 시작 col + 현재 블록 폭)
Y_현재 = Y_다음 - 15.9 × (유형 이동 factor + gap)
```

- 좌측 이동: Y값 감소
- 우측 이동: Y값 증가 (동일 이동량)

## 5. 미구현 (추후)

- **step ↔ 위치 매핑**: 에러 복구를 위한 step 번호 ↔ row/col 역추적
- **/sequence/recover**: 실패 step 기준 상위 블록 제거 → 재배치 큐 생성
- **/sequence/resume**: 복구 성공 후 잔여 블록 큐 반환
